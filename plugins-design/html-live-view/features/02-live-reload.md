# 实时更新

## 功能描述

源码编辑器内容变更后，自动更新渲染视图中的页面，支持可配置的防抖延迟。

## 为什么做？

实时更新是 HTML 编辑器内预览的核心价值。没有实时更新，用户仍需手动刷新，编辑体验退化为"在外部浏览器中按 F5"。

## 收益

- 编辑即见效果，消除手动刷新的中断
- 防抖机制避免频繁重渲染导致页面闪烁
- 可配置延迟适应不同编辑节奏

## 基础设施支持情况

- 现有依赖：Pulsar TextEditor `onDidChange` 事件
- 现有代码：核心渲染（Feature 01）
- 暂不需要新增第三方依赖

## 实现要点

- 监听 TextEditor 的 `onDidChange` 事件
- 防抖延迟默认 300ms，可通过 `html-live-view.debounceDelay` 配置
- 更新时重写 `iframe.srcdoc`（完整替换，非增量）
- 支持 `autoReload` 开关：关闭时需手动刷新

## 风险与注意事项

- srcdoc 完整替换会导致 iframe 内 JS 状态丢失（如输入框内容、动画状态）
- 大文件频繁更新可能导致页面闪烁
- 防抖延迟太长影响体验，太短影响性能
