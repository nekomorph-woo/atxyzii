# 图片资产管理与可靠本地化

## 功能描述

增强 Markdown 图片工作流：支持可靠粘贴/拖拽/截图图片，将图片自动落盘到可配置的 assets 目录；支持相对路径预览解析；在插件配置或管理面板中罗列图片资产、预览大图、查看引用文档、清理未引用图片、重命名图片并同步更新 Markdown 引用。

## 为什么做？

当前图片能力已经能把带文件路径的粘贴/拖拽图片复制到当前文档旁的 `assets/` 目录，但还存在几个明显缺口：

- 截图或浏览器复制出来的 clipboard blob 不一定有 `file.path`，无法稳定本地化。
- 图片预览直接使用 Markdown `src`，相对路径在某些环境下可能无法正确解析到当前文档目录。
- 图片复制到本地后缺少资产管理，时间久了会产生未引用图片、重复图片、错误路径和无法追踪来源的问题。
- 不同项目对图片存放位置的偏好不同，需要允许用户自定义 assets 目录。

## 收益

- 让截图、复制网页图片、拖拽本地图片都能稳定变成本地 Markdown 资源。
- 减少坏图、绝对路径泄漏和文档移动后的路径失效。
- 提供轻量资产管理能力，适合个人知识库、设计文档、README 和 PRD 长期维护。
- 为后续图片上传、发布站点、导出 HTML/PDF 等能力预留统一的资源层。

## 基础设施支持情况

- 现有依赖：Milkdown / ProseMirror image node、当前图片 NodeView、paste/drop handler、Node `fs/path/os`、Pulsar config API。
- 可能新增代码：资产目录配置、剪贴板图片 blob 落盘、相对路径解析器、图片引用索引、资产管理 view/modal、重命名事务、未引用资源扫描。
- 暂不需要新增第三方依赖；如后续需要生成缩略图或读取图片尺寸，可再评估 `sharp`、`image-size` 等依赖，但首版不建议引入。

## 默认 assets 目录策略

- 提供插件配置项 `assetsDirectory`，用户可填写绝对路径或以 `~` 开头的路径。
- 未配置时，默认使用用户 Home 下的系统图片目录：
  - macOS：`~/Pictures/md-wysiwyg-assets`
  - Windows：`%USERPROFILE%\Pictures\md-wysiwyg-assets`
  - Linux / 其他 Unix：优先 `~/Pictures/md-wysiwyg-assets`；如果 `~/Pictures` 不存在，则回退到 `~/md-wysiwyg-assets`
- Markdown 中始终优先写入“相对当前文档”的路径；只有 assets 目录无法相对化或位于不同盘符时，才保守写入绝对路径并提示用户。
- 保存文件名时使用安全文件名：保留扩展名，空格和特殊字符转换为 `-`，重名时追加 `-1`、`-2`。

## 实现要点

- 图片本地化：
  - 支持拖拽本地图片文件。
  - 支持从文件管理器复制图片文件后粘贴。
  - 支持截图后直接粘贴 clipboard image blob，自动写入 `.png`。
  - 支持从浏览器复制图片时，优先读取 image blob；如果只有 URL，则询问或提供“下载为本地资源”的入口。

- 相对路径预览解析：
  - NodeView 渲染图片时，将相对路径按当前 Markdown 文件所在目录解析为 `file://` URL。
  - `src` 仍保持 Markdown 原文，避免保存时被污染。
  - 支持 `./`、`../`、`assets/foo.png`、绝对路径、`file://`、`http(s)://`。

- 资产管理：
  - 在插件设置或命令入口中提供图片资产管理面板。
  - 罗列 assets 目录下图片，展示文件名、大小、修改时间、引用次数。
  - 点击图片可打开预览弹窗，支持查看大图和复制 Markdown 引用。
  - 扫描当前 workspace / 当前项目下 Markdown 文件，建立图片到引用文档的映射。
  - 支持查看某张图片被哪些 Markdown 文档引用，并点击跳转。
  - 支持清理未引用图片；删除前必须展示确认列表。
  - 支持重命名图片，并同步更新所有引用它的 Markdown 文档。

## 风险与注意事项

- 删除和重命名真实文件属于高风险操作，必须二次确认，并尽量支持撤销或至少展示操作摘要。
- “未引用”需要明确扫描范围：首版建议扫描当前 Pulsar project 下的 Markdown 文件，不跨整个 Home。
- Windows 不同盘符无法生成相对路径时要优雅降级。
- 剪贴板 blob 在不同 Electron/Pulsar 版本中能力可能不同，需要分别测试 macOS、Windows、Linux。
- 重命名时要处理 URL 编码、空格、中文文件名和大小写敏感文件系统差异。
