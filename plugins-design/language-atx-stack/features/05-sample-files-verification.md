# 示例文件与高亮验证体系

## 功能描述

在 `grammars/samples/` 目录下为每种接入语言（Kotlin、Dart、C++）创建最小示例文件，涵盖该语言的核心语法构造。建立手动验证流程，确保 grammar 接入后高亮效果正确。

## 为什么做？

Grammar 文件是声明式的正则规则，无法通过单元测试验证渲染效果。需要通过人工打开示例文件、检查 scope 是否正确应用来验证 grammar 质量。示例文件同时作为 grammar 回归测试的基准 — 后续 grammar 更新时可快速对比效果。

## 收益

- 提供可复现的验证流程，降低 grammar 接入后的回归风险
- 示例文件覆盖核心语法构造（关键字、字符串、注释、函数、类），暴露 grammar 的常见缺陷
- 新贡献者可通过示例文件快速理解本包支持的语言和语法覆盖范围

## 基础设施支持情况

- 现有代码：`grammars/` 目录已预留
- 可能新增代码：`grammars/samples/` 目录及每种语言的示例文件
- 暂不需要新增第三方依赖

## 实现要点

### 示例文件要求

每种语言的示例文件需覆盖以下语法构造：

| 构造类别 | 具体内容 |
|----------|----------|
| 关键字 | 语言核心关键字（`fun`、`class`、`if`、`return` 等） |
| 字符串 | 单引号、双引号、多行字符串、字符串插值（若适用） |
| 注释 | 单行注释、多行/文档注释 |
| 函数/方法 | 函数声明、参数、返回值类型 |
| 类型/类 | 类声明、字段、继承、接口 |
| 运算符 | 算术、比较、逻辑运算符 |
| 特殊构造 | 语言特有语法（如 Kotlin data class、Dart null safety、C++ 模板） |

### 文件命名

```
grammars/samples/
  hello.kt       # Kotlin 示例
  hello.dart     # Dart 示例
  hello.cpp      # C++ 示例
```

### 手动验证流程

1. 在 Pulsar 中打开示例文件
2. `Cmd+Alt+Shift+P`（Log Cursor Scope）查看光标位置 scope 栈
3. 确认关键字被标记为 `keyword`、字符串被标记为 `string`、注释被标记为 `comment` 等
4. 若 scope 不正确，检查 grammar 规则并修复

## 风险与注意事项

- 验证依赖人工判断，无法自动化 — 后续可探索 Pulsar headless 模式 + scope 检查的 CI 方案
- 示例文件应保持最小化（< 50 行），覆盖核心构造即可，避免变成语言教程
- Pulsar 的 syntax theme 决定具体颜色，本包只负责生成正确的 scope name
