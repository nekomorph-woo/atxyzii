# 引用搜索

## 功能描述

基于 `rg` 搜索当前符号的引用位置，并以候选列表展示。

## 为什么做？

跳到定义之后，用户常常还想知道这个符号在哪里被调用。完整引用分析需要语义能力，但纯文本引用搜索已经能覆盖大量阅读场景。

## 收益

- 提供轻量“Find References”体验。
- 适合 AI Coding 前快速评估影响范围。
- 复用 `rg`、候选列表、路径排序能力。
- 不引入 LSP 也能先做出实用能力。

## 基础设施支持情况

- 依赖前置：SearchProvider、CandidateListView、CandidateRanker。
- 可能新增命令：`code-hopper:find-references`。
- 可能使用 `rg --word-regexp` 或语言相关边界正则。

## 实现要点

- 从当前 token 搜索全项目引用。
- 默认排除定义行可以后续处理，初版可显示全部命中。
- 候选展示行文本片段，方便判断调用上下文。
- 当前文件、Git modified、同目录优先。
- 最大结果数可配置。

## 风险与注意事项

- 文本引用会包含注释、字符串、同名局部变量等噪音。
- 不应承诺语义精确引用。
- 大仓库必须设置超时和结果上限。
