# 插件开发变更隔离

会话初始确定开发/设计某个指定插件时，后续变更严格限制在该插件范围内：

| 📂 允许变更 | 用途 |
|-------------|------|
| `tk-ATx/<plugin-name>/` | 插件源码，代码改动和 commit 在此 submodule 仓库内执行 |
| `plugins-design/<plugin-name>/` | 设计文档（PRD、技术方案、计划等） |

| 🚫 不得变更 | 说明 |
|-------------|------|
| 其他 `<submodule>/` | 不得修改其他插件源码 |
| `plugins-design/<其他插件>/` | 不得修改其他插件设计文档 |
| 主仓库配置（`.gitmodules`、`CLAUDE.md` 等） | 除非用户明确要求 |
| `.claude/rules/`、`.claude/skills/` | 除非用户明确要求 |

例外：用户明确指示修改主仓库文件时，可以执行。

验证：每次文件变更前检查路径是否在允许范围内，超出范围时向用户确认。
